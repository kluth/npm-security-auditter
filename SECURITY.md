## Reporting a Vulnerability

We take the security of `auditter` seriously. If you have found a vulnerability in this tool, please do not disclose it publicly until we have had a chance to fix it.

### How to Report

Please submit a report via email to **security@example.com** (replace with actual if available) or use the GitHub Security Advisory reporting feature if enabled.

In your report, please include:

1.  **Type of Vulnerability:** (e.g., Buffer Overflow, RCE, Information Disclosure)
2.  **Full Description:** A clear explanation of the vulnerability.
3.  **Steps to Reproduce:** Detailed steps or a proof-of-concept script.
4.  **Impact:** What can an attacker achieve with this vulnerability?

### Response Timeline

*   **Acknowledgment:** We aim to acknowledge reports within 48 hours.
*   **Assessment:** We will assess the severity and impact within 5 business days.
*   **Fix:** A fix will be developed and released as soon as possible, prioritizing critical issues.

Thank you for helping keep the ecosystem safe!
